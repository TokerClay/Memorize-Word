# 统计模块

"我的"页面的数据计算和 UI 渲染，包括打卡日历、进度条和生词本标签。

## 结构

```
index.html 内逻辑分区 (行 678-757):
├── getTrackedSwiped()          # 按 trackMode 返回对应刷词数 (行 678-683)
├── getTrackedRounds()          # 按 trackMode 返回对应轮数 (行 684-689)
├── updateMyPage()              # 渲染所有统计指标和进度条 (行 692-714)
├── renderCalendar()            # 28 天打卡日历 (行 716-732)
├── renderBookmarkTags()        # 生词本标签（最近 20 词）(行 734-741)
├── switchPage()                # 页面切换 + tab 高亮 (行 744-757)
├── renderFullCalendar()        # 完整月历弹窗 (行 951-965)
└── 日历事件绑定                 # 点击翻月/关闭 (行 942-949)
```

## 关键文件

| 函数 | 目的 |
|------|------|
| `updateMyPage()` | 核心渲染函数：今日数、剩余数、轮数、天数、进度条、日历、生词本 |
| `renderCalendar()` | 生成最近 4 周的打卡日历 HTML，通过 `innerHTML` 渲染 |
| `renderBookmarkTags()` | 生成生词本标签 HTML，最多显示 20 个 + 省略号 |
| `renderFullCalendar()` | 生成完整月历 HTML（6 行 x 7 列），支持翻月 |
| `getTrackedSwiped/Rounds()` | 根据 `settings.trackMode` 返回当前追踪模式的数据 |

## 依赖

**本模块依赖**:
- 数据层（`data` 对象, `settings` 对象）
- 词库系统（`TOTAL`, `BANKS[currentBank].name`, `words`）

**依赖本模块的**:
- `switchPage('my')` 触发 `updateMyPage()`
- `nextWord()` 每次刷词后触发 `updateMyPage()`

## 规范

### 渲染时机

`updateMyPage()` 在以下场景调用：
- 切换到"我的"页面
- 每次刷词完成后（`nextWord()` 末尾）
- 设置变更后
- 词库切换完成后
- 目标设置弹窗关闭后

### 目标达标高亮

当 `todaySwiped >= dailyGoal` 时，`#goalItem` 添加 `.done` class，该元素显示浅绿背景。

### 打卡日历视觉

| 状态 | CSS class | 样式 |
|------|-----------|------|
| 普通 | `cal-day` | 灰色背景 |
| 已打卡 | `cal-day checked` | 绿色圆点背景 |
| 今天 | `cal-day today` | 蓝色边框 |
| 未来日期 | — | `color: transparent` 文字透明 |
