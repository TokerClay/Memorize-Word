# 刷词模块

单词循环展示、用户交互处理和进度保存。

## 结构

```
index.html 内逻辑分区 (行 541-675):
├── startStudy(mode)            # 启动刷词，初始化列表/进度 (行 542-563)
├── currentStudyList()          # 根据 studyMode 返回当前列表 (行 566-568)
├── showStudyWord()             # 渲染当前单词到 DOM (行 571-579)
├── updateBookmarkBtn()         # 更新收藏按钮状态 (行 582-589)
├── toggleBookmark()            # 添加/移除收藏 (行 591-599)
├── vibrate()                   # 触觉反馈 (行 602)
├── prevWordBusy / prevWord()    # 上一词 + 互斥锁 (行 604-617)
├── nextWordBusy / nextWord()    # 下一词 + 互斥锁 (行 619-656)
├── getModeLabel()              # 模式中文标签 (行 658)
├── saveProgress()              # 保存当前 progress 到 data (行 660-665)
└── speakWord()                 # Web Speech API 朗读 (行 668-675)
```

## 关键文件

| 函数 | 目的 |
|------|------|
| `startStudy()` | 入口：准备单词列表、恢复进度、渲染首词、切换页面 |
| `nextWord()` | 前进游标，累加统计，检测轮次完成和每日目标 |
| `prevWord()` | 后退游标，保存进度（非 bookmark 模式） |
| `showStudyWord()` | 将 `list[index]` 的属性写入 DOM（word, pos, meaning, phonetic） |
| `toggleBookmark()` | 在 `data.bookmarks[]` 中添加/移除单词 |

## 依赖

**本模块依赖**:
- 词库系统（`words[]`, `ordered[]`, `shuffled[]`, `bookmarkWords[]`, `wordIndex`）
- 数据层（`data` 对象, `saveData()`, `saveSettings()`）

**依赖本模块的**:
- 事件系统（触摸/点击/键盘事件绑定）
- 首页按钮事件

## 规范

### 并发保护

每个异步操作前检查互斥锁：

```javascript
if (nextWordBusy) return;
nextWordBusy = true;
try {
    // 异步操作
} catch (e) {
    console.warn('nextWord error:', e);
}
nextWordBusy = false;
```

### 轮次完成

非 bookmark 模式轮次完成后：
- `data.rounds++`
- 显示完成弹窗
- `index = 0`
- 保存进度但不累计 `todaySwiped`（避免新轮次首词被计数）
- 返回（不执行计数逻辑）

### 进度持久化时机

- 非 bookmark 模式：每次 `nextWord()` 和 `prevWord()` 都保存
- Bookmark 模式：不自动保存进度，仅在轮次完成时保存

### 错误处理

- `prevWord()` 用 try/catch 包裹核心逻辑
- `nextWord()` 用 try/catch 包裹整个异步流程
- 异常时 `console.warn` 记录，互斥锁正常释放
