# 词库系统

词库系统负责多词库的管理、切换和运行时的单词数据供应。每个词库的刷词进度和设置完全独立。

## 什么是词库？

词库是包含一类 CET 考试核心词汇的 JSON 数据集。目前支持 CET-4（2607 词）和 CET-6（548 词）。每个词库有一个唯一 key（如 `cet4`）和对应的 JSON 文件路径。

## 数据隔离

每个词库使用独立的 IndexedDB key 存储数据：

```
data:{uid}:cet4    ← CET-4 的 StudyData
data:{uid}:cet6    ← CET-6 的 StudyData
settings:{uid}:cet4  ← CET-4 的设置
settings:{uid}:cet6  ← CET-6 的设置
```

切换词库时，当前进度写入旧 key，然后从新 key 读取新词库的独立数据。

## 不变量

1. **词库名来源唯一**: 词库名称只能来自 `BANKS[bank].name`，通过 `escapeHTML()` 转义后用于 `innerHTML`
2. **词库切换先保存**: `switchBank()` 始终先 `await saveData()` 再切换
3. **词库切换有幂等保护**: 切换到当前词库且 `words.length > 0` 时直接 return
4. **切换时切回首页**: `switchPage('home')` 防止学习页显示旧词库数据
5. **活跃词库持久化**: `activeBank` 字段保存在 `profile` 中，刷新后自动恢复
6. **新用户必须选择词库**: `!uid` 时 `bankSelectModal` 点击遮罩不会关闭，强制选择

## 词库加载流程

```mermaid
sequenceDiagram
    participant F as fetch()
    participant L as loadWords
    participant UI as UI

    L->>UI: 显示 loading 遮罩
    L->>F: fetch(BANKS[bank].file)
    F-->>L: JSON 数据
    L->>L: words = data; TOTAL = words.length
    L->>L: ordered = sorted(words)
    L->>L: buildWordIndex()
    L->>UI: 隐藏 loading 遮罩
    L->>UI: showHomeWord() + updateMyPage()
```

如果 fetch 失败，遮罩关闭后首页显示"加载失败，请刷新重试"。

## 添加新词库

在 `BANKS` 对象中注册：

```javascript
var BANKS = {
    cet4: {name: '四级核心词汇', file: 'words_cet4.json'},
    cet6: {name: '六级核心词汇', file: 'words_cet6.json'},
    ielts: {name: '雅思核心词汇', file: 'words_ielts.json'}  // 新增
};
```

创建对应的 JSON 文件并部署到同一域名下。词库选择弹窗自动渲染所有 BANKS 条目，无需修改 UI 代码。
