# 数据层

IndexedDB 数据库初始化、读写操作和跨版本数据迁移。

## 结构

```
index.html 内逻辑分区:
├── DB_NAME / DB_VERSION        # 数据库配置 (行 379)
├── openDB()                    # 打开/升级数据库 (行 391-406)
├── dbGet(store, key)           # 读取键值 (行 408-414)
├── dbPut(store, key, value)    # 写入键值 (行 416-422)
├── generateUID()               # 生成唯一用户ID (行 425)
├── migrateV1toV2()             # v1→v2 全局→用户隔离 (行 427-442)
├── migrateKeys()               # v2→v3 词库隔离+主题独立 (行 444-462)
├── data / settings / username  # 全局状态变量 (行 465)
├── dataKey() / settingsKey()   # 构建 key 字符串 (行 467-468)
├── loadAll()                   # 加载用户数据到内存 (行 470-479)
├── saveData()                  # 保存 StudyData (行 481)
├── saveSettings()              # 保存 UserSettings (行 482)
└── saveProfile()               # 保存 UserProfile + active (行 483)
```

## 关键文件

| 函数 | 目的 |
|------|------|
| `openDB()` | 创建/升级 IndexedDB，触发 `onupgradeneeded` 迁移标志 |
| `dbGet()` | Promise 包装的读操作，失败返回 null |
| `dbPut()` | Promise 包装的写操作，通过 `tx.oncomplete` 确保写入完成 |
| `migrateKeys()` | v2→v3 迁移，复制数据到按词库隔离的 key，提取主题字段 |
| `loadAll()` | 初始化时读取 uid + data + settings 到内存 |

## 依赖

**本模块依赖**: IndexedDB API (浏览器内置)

**依赖本模块的**: 所有业务模块（词库系统、刷词引擎、统计系统、主题系统）

## 规范

### 错误处理

- 读操作失败：`resolve(null)`，调用方用 `|| defaultData()` 回退
- 写操作失败：`console.warn()` 记录，调用方 try/catch 捕获
- 迁移失败：静默放弃，不影响主流程

### Key 命名约定

```
profile:active          — 当前活跃用户的 uid
profile:user:{uid}      — 用户档案
data:data:{uid}:{bank}  — 学习数据
settings:settings:{uid}:{bank}  — 设置
settings:theme:{uid}    — 主题偏好
```

### 版本迁移

- 使用 `e.oldVersion` 判断是否需要迁移
- 迁移逻辑只在 `onupgradeneeded` 中触发
- 迁移标志在函数执行后立即置 false，防止重复执行

## 添加新 Store

1. 递增 `DB_VERSION`
2. 在 `onupgradeneeded` 中添加 `d.createObjectStore('newStore', {keyPath:'key'})`
3. 添加条件 `if(e.oldVersion < NEW_VERSION)` 包裹

## 添加新迁移

1. 在 `loadAll()` 之后调用新迁移函数
2. 使用 `migrateKey` 风格标志控制执行
3. 迁移后更新内存中的对应变量
