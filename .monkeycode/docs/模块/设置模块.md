# 设置模块

每日目标、追踪模式、TTS 开关和主题偏好的配置与持久化。

## 结构

```
index.html 内逻辑分区:
├── Settings UI
│   ├── updateSettingsUI()          # 刷新追踪模式按钮/TTS开关状态 (行 760-765)
│   ├── #settingsBtn click          # 打开设置弹窗 (行 896-899)
│   ├── #modalClose click           # 关闭设置弹窗 (行 901)
│   ├── .modal-option click         # 切换追踪模式 (行 903-905)
│   └── #ttsToggle click            # 切换 TTS 开关 (行 906-908)
├── Goal Picker
│   ├── goalValues[]                # 可选目标值 10–300 步长 5 (行 768-769)
│   ├── renderGoalPicker()          # 渲染滚动选择器 (行 771-777)
│   ├── setGoalPickerPos()          # 定位选项到当前值 (行 779-785)
│   └── bindGoalPicker()            # 绑定触摸/鼠标拖拽事件 (行 787-818)
├── Goal picker instances
│   ├── bindGoalPicker('goalPickerContainer', ...)   # 设置弹窗的目标选择器 (行 910-912)
│   └── bindGoalPicker('goalPickerContainer2', ...)  # 首次使用的目标选择器 (行 914-916)
├── First-time goal modal
│   ├── renderGoalPicker('goalPickerContainer2', ...) # 新用户目标弹窗 (行 923)
│   ├── #goalModalClose click        # 确认目标 (行 924-926)
│   └── #goalModal overlay click     # 遮罩关闭时存默认值 50 (行 928-937)
└── Theme
    ├── themeExplicitlySet flag      # 是否手动切换过主题 (行 885)
    ├── saveTheme(t)                 # 持久化主题到 IndexedDB (行 886)
    ├── #themeBtn click              # 切换深色/浅色 (行 887-893)
    └── matchMedia listener          # 跟随系统主题（仅当未手动设置）(行 1087-1091)
```

## 关键文件

| 函数/元素 | 目的 |
|----------|------|
| `bindGoalPicker()` | 目标值滚动选择器：支持触摸拖拽和鼠标拖拽，松手时吸附到最近值 |
| `renderGoalPicker()` | 生成 10–300 步长 5 的垂直滚动列表 |
| `#themeBtn` | 深色/浅色一键切换，保存到 `theme:{uid}` |
| `themeExplicitlySet` | 标记用户是否手动切换过主题，控制是否继续跟随系统 |

## 依赖

**本模块依赖**:
- 数据层（`settings` 对象, `saveSettings()`, `saveTheme()`）
- 数据持久化层（`dbPut` 写 theme）

**依赖本模块的**:
- 统计模块（`updateMyPage()` 读取 `settings.*` 进行渲染）
- 刷词模块（`speakWord()` 检查 `settings.ttsEnabled`）
- DOM (CSS Variables 通过 `.dark` class 控制)

## 规范

### 目标选择器

- 可选值范围: 10–300，步长 5
- 通过 CSS `transform: translateY()` 实现平滑滚动
- 触摸/鼠标 dual bind：`touchstart/move/end` + `mousedown/move/up`
- 松手时 `Math.round()` 吸附到最近值

### 主题

- 存储路径: `IndexedDB.settings/theme:{uid}`（全局，不随词库切换）
- 阻塞脚本: `<head>` 中同步检测 `prefers-color-scheme`，先于 CSS 渲染设置 `.dark`
- 自动跟随: `themeExplicitlySet=false` 时，OS 主题变化自动切换
- 手动优先: 用户点击主题按钮后 `themeExplicitlySet=true`，不再跟随系统

### 新用户首次设置

- 用户名 → 词库选择 → 目标弹窗
- 遮罩关闭也自动存档（默认值 50），防止用户跳过目标设置导致未初始化状态

### 持久化时机

- 各项设置变更时立即 `await saveSettings()`
- 主题变更时 `await saveTheme()`
- 异步写入，不阻断 UI 响应
